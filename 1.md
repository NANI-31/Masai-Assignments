## Part 1: The Journey - Eliza to ChatGPT

### **966: ELIZA - The First Chatbot**

**How ELIZA worked:**

```
User: ""I feel sad today""
ELIZA: ""Why do you feel sad today?""
```

**Logic:**

```
IF input contains ""I feel [emotion]""
THEN respond ""Why do you feel [emotion]?""
```

**Simple pattern matching:**

- Rule-based system
- No understanding
- Just templates and keyword matching

```py
# Simplified ELIZA logic
def eliza_response(user_input):
    if ""I feel"" in user_input:
        emotion = extract_emotion(user_input)
        return f""Why do you feel {emotion}?""
    elif ""my mother"" in user_input:
        return ""Tell me more about your mother""
    else:
        return ""Please go on""
```

Limitations:

- No context understanding
- Easy to break
- Purely deterministic (same input = same output)

---

### **2022: ChatGPT - Probabilistic Reasoning**

**How ChatGPT works:**

```
User: ""I feel sad today""
ChatGPT: ""I'm sorry to hear you're feeling sad. Would you like to talk about
what's bothering you? Sometimes sharing helps. I'm here to listen.""
```

**Logic:**

- Analyzes context
- Understands emotion
- Generates human-like empathetic response
- Considers thousands of possible replies, picks most probable

**Key differences:**

| ELIZA (1966)     | ChatGPT (2022)          |
| ---------------- | ----------------------- |
| If/Else rules    | Neural networks         |
| Pattern matching | Probabilistic reasoning |
| No context       | 128K+ token context     |
| Deterministic    | Creative/varied         |
| Breaks easily    | Handles ambiguity       |

---

### **The Evolution**

```
1966: ELIZA
↓
1990s: Expert Systems (more complex rules)
↓
2000s: Statistical ML (learning from data)
↓
2017: Transformers (attention mechanism)
↓
2022: ChatGPT (GPT-3.5)
↓
2023: GPT-4, Claude 3, Gemini 1.5
↓
2024: Multimodal AI (text + images + code)
```

**What changed everything:**

- Transformers (2017) - ""Attention is All You Need"" paper
- Scale - Billions of parameters
- Training data - Entire internet
- Reinforcement Learning from Human Feedback (RLHF)

## **Part 2: Prompting - The New Superpower**

### **Why Prompting Matters**

**Bad Prompt = Bad Output**

❌ Simple Prompt:

```
"Write a recipe"
```

**Output:**

```
"Here's a simple pasta recipe:

1. Boil water
2. Cook pasta
3. Add sauce
   Done!""
```

**Good Prompt = Great Output**

✅ Well-Framed Prompt:

```
""You are a Michelin-star chef specializing in Italian cuisine. Create a
gourmet pasta recipe using seasonal ingredients. Include:

- Fancy dish name
- Ingredient list with precise measurements
- Step-by-step cooking instructions with timing
- Plating suggestions
- Wine pairing recommendation
  Target audience: Food enthusiasts comfortable with intermediate cooking.""
```

**Output:**

```
[Detailed, professional recipe with Italian flair]
```

### **The CO-STAR Framework (Advanced)**

| C= Context    | ""I'm launching a SaaS product for remote teams""       |
| ------------- | ------------------------------------------------------- |
| O = Objective | ""Generate 10 feature ideas""                           |
| S = Style     | ""Bullet points, concise""                              |
| T = Tone      | ""Innovative but realistic""                            |
| A = Audience  | ""Product managers""                                    |
| R = Response  | ""Table: Feature / Value Prop / Effort (Low/Med/High)"" |

Example CO-STAR Prompt:

```
Context: I'm building a meal planning app for busy professionals who want
to eat healthy but have limited time.

Objective: Generate a unique value proposition that differentiates us from
existing apps like Mealime or Eat This Much.

Style: Marketing copy, punchy and memorable

Tone: Empowering, not preachy. Focus on time-saving, not guilt.

Audience: 25-40 year old professionals, tech-savvy, health-conscious


Response:

1. One-liner tagline (max 10 words)
2. Three-sentence value proposition
3. Two key features that support the value prop
```

## **Context Windows Explained**

**What is a context window?** The amount of text the model can ""remember"" at once.

**Small context (8K tokens):**

- Can process ~6,000 words
- Good for: Single conversations, short docs

**Medium context (128K tokens):**

- Can process ~96,000 words
- Good for: Entire books, long conversations

**Large context (1M tokens):**

- Can process ~750,000 words
- Good for: Entire codebases, multiple books

```
Token calculation:

Rough estimate:
1 token ≈ 0.75 words (English)

Examples:

- ""Hello, world!"" = ~3 tokens
- This paragraph = ~50 tokens
- A novel = ~100,000 tokens
- Entire Harry Potter series = ~1M tokens
```

**Why it matters:**

```
# Small context problem

prompt = ""Summarize this 50-page document""

# Error if document > context window

# Solution with large context

prompt = ""Here's a 200-page codebase: [paste entire repo]
Find all security vulnerabilities""

# Works with Gemini 1.5 (1M context)
```

#### **API Best Practices**

**1. Environment Variables**

```js

❌ const API_KEY = 'AIzaSyC...'

# Use environment variables
✅ const API_KEY = process.env.GEMINI_API_KEY 2. Error Handling
```

**2. Error Handling**

```js

async function safeGeminiCall(prompt) {
  try {
    const response = await model.generateContent(prompt);
    return response.text();
  } catch (error) {
    if (error.message.includes('quota')) {
      return ""Rate limit exceeded. Try again in a minute."";
    }
    return ""Error: "" + error.message;
  }
}
```

**3. Rate Limiting**

```js
// Simple rate limiter
let lastCall = 0;
const MIN_INTERVAL = 4000; // 4 seconds = 15 calls/min

async function rateLimitedCall(prompt) {
  const now = Date.now();
  const timeSinceLastCall = now - lastCall;

  if (timeSinceLastCall < MIN_INTERVAL) {
    await new Promise((resolve) =>
      setTimeout(resolve, MIN_INTERVAL - timeSinceLastCall),
    );
  }

  lastCall = Date.now();
  return await model.generateContent(prompt);
}
```

    </div>
    <div class=""input-group"">
      <label>Ingredient 3</label>
      <input type=""text"" id=""ingredient3"" placeholder=""e.g., Garlic"">
    </div>

    <button id=""cookBtn"">Cook Magic ✨</button>
    <div class=""loading"">🔮 Creating your masterpiece...</div>
    <div class=""error""></div>

    <div class=""result-card"" id=""resultCard"">
      <div class=""dish-name"" id=""dishName""></div>
      <ol class=""recipe-steps"" id=""recipeSteps""></ol>
    </div>

  </div>

  <script>
    const API_KEY = 'YOUR_GEMINI_API_KEY'; // Replace with your key
    const API_URL = `https://generativelanguage.googleapis.com/v1beta/models/gemini-1.5-flash:generateContent?key=${API_KEY}`;

    document.getElementById('cookBtn').addEventListener('click', async () => {
      // Get ingredients
      const ing1 = document.getElementById('ingredient1').value.trim();
      const ing2 = document.getElementById('ingredient2').value.trim();
      const ing3 = document.getElementById('ingredient3').value.trim();

      // Validate
      if (!ing1 || !ing2 || !ing3) {
        showError('Please enter all 3 ingredients');
        return;
      }

      // UI states
      const btn = document.getElementById('cookBtn');
      const loading = document.querySelector('.loading');
      const resultCard = document.getElementById('resultCard');
      const errorDiv = document.querySelector('.error');

      btn.disabled = true;
      loading.style.display = 'block';
      resultCard.style.display = 'none';
      errorDiv.style.display = 'none';

      try {
        // Call Gemini API
        const response = await fetch(API_URL, {
          method: 'POST',
          headers: { 'Content-Type': 'application/json' },
          body: JSON.stringify({
            contents: [{
              parts: [{
                text: `You are a creative Michelin-star chef. Given these 3 ingredients: ${ing1}, ${ing2}, ${ing3}, create:
1. A fancy French-inspired dish name
2. Exactly 3 cooking steps (each step should be one clear sentence)

Respond ONLY with valid JSON in this exact format:
{
  ""dishName"": ""Le Poulet Citronné aux Herbes"",
  ""steps"": [
    ""Season the chicken with minced garlic and lemon zest, then sear in olive oil until golden."",
    ""Deglaze the pan with lemon juice and white wine, creating a silky sauce."",
    ""Garnish with fresh herbs and serve immediately with roasted vegetables.""
  ]
}

Important: Return only the JSON, no markdown, no explanation.`
              }]
            }]
          })
        });

        if (!response.ok) {
          throw new Error('API request failed');
        }

        const data = await response.json();
        const text = data.candidates[0].content.parts[0].text;

        // Parse JSON (handle potential markdown wrapping)
        let jsonText = text.trim();
        if (jsonText.startsWith('```json')) {
          jsonText = jsonText.replace(/```json\n?/g, '').replace(/```/g, '');
        }

        const recipe = JSON.parse(jsonText);

        // Display result
        document.getElementById('dishName').textContent = recipe.dishName;
        const stepsList = document.getElementById('recipeSteps');
        stepsList.innerHTML = '';

        recipe.steps.forEach((step, index) => {
          const li = document.createElement('li');
          li.textContent = `${index + 1}. ${step}`;
          stepsList.appendChild(li);
        });

        resultCard.style.display = 'block';

      } catch (error) {
        showError('Oops! The kitchen is closed. Try again.');
        console.error(error);
      } finally {
        btn.disabled = false;
        loading.style.display = 'none';
      }
    });

    function showError(message) {
      const errorDiv = document.querySelector('.error');
      errorDiv.textContent = message;
      errorDiv.style.display = 'block';
    }
  </script>
